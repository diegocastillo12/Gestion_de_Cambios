# Matriz de Trazabilidad de Requisitos y Pruebas de Aceptación

Esta matriz relaciona los Requisitos No Funcionales (RNF) con los Requisitos Funcionales (RF) del sistema GestioCambios, detallando sus categorías, atributos de calidad, disponibilidad y las respectivas pruebas de aceptación con sus descripciones de prueba.

---

## 1. Cuadro de Relación y Pruebas de Aceptación

| ID RNF | ID RF Asociado | Categoría | Prueba de Aceptación | Descripción de la Prueba | Atributo de Calidad | Disponibilidad |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| RNF-01 | RF-01, RF-16 | Seguridad | PA-SEC-01 (Verificación de Cifrado en BD y Login) | Registrar un usuario global y validar directamente en la base de datos MySQL que la contraseña almacenada en la columna `password_hash` esté cifrada usando Bcrypt (iniciando con $2a$). Probar el inicio de sesión con dicha contraseña para validar el descifrado correcto. | Confidencialidad / Integridad | Alta (Uso obligatorio continuo) |
| RNF-02 | RF-03, RF-09, RF-11, RF-14 | Base de Datos | PA-PERF-01 (Carga de Conexiones Simultáneas) | Ejecutar un script de pruebas automatizadas que dispare 50 consultas de lectura concurrentes sobre la API de tickets (`/api/tickets`) y verificar que el pool de conexiones en `db.js` reutilice las conexiones activas sin agotar la capacidad de MySQL ni lanzar errores de Timeout. | Eficiencia / Escalabilidad | Alta (99.9% disponibilidad del pool) |
| RNF-03 | Todos los RF | Arquitectura | PA-ARCH-01 (Inspección de Independencia MVC) | Analizar de forma estática la estructura del código para verificar que las vistas EJS no contengan consultas SQL directas, y que toda petición HTTP sea derivada exclusivamente a través de los enrutadores (`routes/`) hacia controladores (`controllers/`) y luego a los modelos (`models/`). | Mantenibilidad / Modificabilidad | Alta (100% de la arquitectura) |
| RNF-04 | Todos los RF | Portabilidad | PA-COMP-01 (Despliegue y Arranque Limpio) | Descargar el repositorio del proyecto en un servidor limpio con Node.js v18+ y MySQL 8.0, ejecutar las tareas de inicialización (`npm install`, `node seed.js` y `npm run dev`) y verificar que el servidor levante sin conflictos de dependencias ni fallos de sintaxis en consola. | Portabilidad / Compatibilidad | Alta (Compatibilidad multiplataforma) |
| RNF-05 | RF-03, RF-10, RF-11 | Usabilidad | PA-USER-01 (Prueba de Visualización Responsiva) | Redimensionar la resolución del navegador a resoluciones de dispositivos móviles (375px) y tablets (768px). Comprobar visualmente que el menú lateral se colapse de forma responsive, que los textos no se desborden de sus tarjetas y que los listados de tickets sean legibles. | Usabilidad / Adaptabilidad | Media (Disponible en navegadores móviles) |
| RNF-06 | Todos los RF | Confiabilidad | PA-RELI-01 (Intercepción de Excepciones Runtime) | Provocar deliberadamente una excepción en el servidor (ej. ingresando una URL inexistente o una variable corrupta en el detalle de un proyecto) y comprobar que el middleware de manejo de errores intercepte la caída, respondiendo con la página controlada `error.ejs` y manteniendo el proceso del servidor activo. | Confiabilidad / Tolerancia a Fallos | Alta (Continuidad de servicio ininterrumpida) |
| RNF-07 | RF-09, RF-11 | Rendimiento | PA-TIME-01 (Medición de Tiempo de Respuesta) | Cargar la base de datos con un volumen de 1000 registros de tickets de cambios e historial. Ejecutar una consulta con filtros combinados desde el panel y verificar en las herramientas de desarrollo del navegador que la respuesta JSON del servidor sea inferior a 2.0 segundos. | Eficiencia de Tiempo | Alta (Tiempos de respuesta óptimos bajo carga) |
